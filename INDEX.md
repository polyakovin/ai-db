## О проекте

Эта база знаний собирает практический опыт применения AI-агентов в реальной работе: паттерны проектирования, архитектурные решения, подбор инструментов, кейсы внедрения, переводы ключевых статей и сравнение платформ. Vault рассчитан на инженеров, технических лидов и всех, кто строит агентные системы руками.

**Цели проекта:**
- Систематизировать знания об агентных архитектурах и паттернах
- Собрать проверенные рецепты (recipes) для типовых задач
- Накопить кейсы из практики и выводы по ним
- Вести глоссарий, исключающий разночтения терминов

## Глоссарий ключевых терминов

| Термин (EN) | Термин (RU) | Определение |
|---|---|---|
| **Agent** | Агент | Автономная система, использующая LLM для восприятия среды, принятия решений и выполнения действий |
| **Tool / Tool Use** | Инструмент / Вызов инструментов | Функция или API, которую агент может вызвать для взаимодействия с внешним миром |
| **Chain** | Цепочка | Последовательность вызовов LLM и инструментов, выполняемая в заданном порядке |
| **Orchestration** | Оркестрация | Координация работы нескольких агентов или компонентов для достижения общей цели |
| **RAG** (Retrieval-Augmented Generation) | Генерация с дополненной выдачей | Подход, при котором LLM дополняет ответ релевантными фрагментами из внешней базы знаний |
| **ReAct** (Reasoning + Acting) | ReAct (Рассуждение + Действие) | Архитектурный паттерн, где агент чередует шаги рассуждения и действия |
| **Guardrails** | Ограждения / Ограничения | Механизмы валидации и фильтрации ввода/вывода агента для обеспечения безопасности и корректности |
| **Human-in-the-Loop (HITL)** | Человек в цикле | Паттерн, при котором человек участвует в проверке или утверждении решений агента |
| **Memory** | Память | Механизм сохранения контекста и истории взаимодействий агента (краткосрочная, долгосрочная, рабочая память) |
| **Prompt** | Промпт / Инструкция | Текстовый запрос или системная инструкция, управляющая поведением LLM |
| **Token** | Токен | Минимальная единица текста, обрабатываемая LLM (~0.75 слова для английского, ~0.5 слова для русского) |
| **Context Window** | Контекстное окно | Максимальный объём токенов, который модель может обработать за один запрос |
| **Embedding** | Эмбеддинг / Векторное представление | Числовой вектор, представляющий семантическое значение текста; используется в RAG и поиске |
| **Hallucination** | Галлюцинация | Генерация моделью фактологически неверной или вымышленной информации |
| **Fine-tuning** | Дообучение / Тонкая настройка | Дополнительное обучение предобученной LLM на специализированном наборе данных |
| **MCP** (Model Context Protocol) | MCP (Протокол контекста модели) | Открытый протокол Anthropic для стандартизованного подключения инструментов и источников данных к LLM |
| **Agentic Workflow** | Агентный воркфлоу | Рабочий процесс, в котором агент автономно принимает решения о последовательности шагов |
| **Multi-Agent System (MAS)** | Многоагентная система | Система из нескольких взаимодействующих агентов, каждый со своей ролью |
| **Planning** | Планирование | Способность агента разбивать сложную задачу на подзадачи и определять порядок их выполнения |
| **Evaluation / Eval** | Оценка / Эвалюация | Систематическая проверка качества и надёжности работы агента на наборе тестовых сценариев |
| **Latency** | Задержка / Латентность | Время от отправки запроса до получения полного ответа от агента |
| **Cost** | Стоимость | Денежные затраты на вызовы LLM API; ключевой параметр при проектировании агентов |

## Как работать с vault

1. **Навигация** — открывайте файлы через Obsidian Graph View или Ctrl/Cmd+O
2. **Связи** — используйте `wikilinks` для связывания заметок; все связи видны на графе
3. **Шаблоны** — для новых заметок в разделах используйте шаблоны из папки `_templates` (будет добавлена)
4. **Поиск** — Ctrl/Cmd+Shift+F для полнотекстового поиска по vault
5. **Git** — коммитьте изменения осмысленно: `раздел: что сделано`
