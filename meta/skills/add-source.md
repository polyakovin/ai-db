# Скилл: Добавление источника в проект

## Назначение
Добавляет новый источник информации в базу знаний с соблюдением структуры проекта и метаданных.

## Поддерживаемые типы источников
- **URL** — веб-страницы, статьи, документация
- **Файл** — локальные документы (PDF, MD, TXT)
- **Документ** — записи из баз данных, заметки

## Команды

### 1. Добавить URL-источник
```bash
/add-source url <URL> [--category <категория>] [--tags <теги>]
```

**Параметры:**
- `URL` — обязательный, адрес источника
- `--category` — опциональный, категория для размещения (по умолчанию: `sources`)
- `--tags` — опциональный, теги через запятую

**Пример:**
```bash
/add-source url https://arxiv.org/abs/2301.07041 --category articles --tags llm,transformer
```

### 2. Добавить файл
```bash
/add-source file <путь_к_файлу> [--category <категория>]
```

**Параметры:**
- `путь_к_файлу` — обязательный, путь к локальному файлу
- `--category` — опциональный, категория для размещения

**Пример:**
```bash
/add-source file ~/Downloads/attention-paper.pdf --category patterns
```

### 3. Добавить документ (заметку)
```bash
/add-source doc <название> [--category <категория>] [--template <шаблон>]
```

**Параметры:**
- `название` — обязательный, название документа
- `--category` — опциональный, категория для размещения
- `--template` — опциональный, шаблон документа (default, article, tutorial)

**Пример:**
```bash
/add-source doc "Конспект: Transformer" --category patterns --template article
```

## Алгоритм работы

### Для URL:
1. Извлечь метаданные (заголовок, описание, автора, дату)
2. Определить тип контента (статья, документация, видео)
3. Создать markdown-файл в указанной категории
4. Заполнить frontmatter метаданными
5. Добавить ссылку на оригинал
6. Создать запись в README.md или обзорной странице раздела

### Для файла:
1. Проверить существование файла
2. Определить тип файла по расширению
3. Скопировать файл в структуру проекта (опционально)
4. Создать markdown-файл с метаданными
5. Добавить запись в README.md или обзорной странице раздела

### Для документа:
1. Создать новый markdown-файл по шаблону
2. Заполнить frontmatter (дата, автор, теги)
3. Добавить заголовок и базовую структуру
4. Обновить README.md или обзорную страницу раздела

## Структура frontmatter
```yaml
---
title: <название источника>
url: <оригинальная ссылка> # для URL
type: <url|file|doc>
category: <категория>
tags: [тег1, тег2]
added: <дата добавления>
status: <new|reviewed|processed>
---
```

## Правило именования
- Новые пользовательские папки и Markdown-файлы создаются на английском в `kebab-case`.
- Обзорная страница раздела всегда называется `overview.md`.
- Не использовать русские имена файлов, пробелы, camelCase, PascalCase или числовые префиксы для сортировки.

## Категории по умолчанию
- `concepts` — базовые понятия и архитектурные блоки
- `patterns` — повторяемые проектные решения
- `tools` — инструменты, модели, SDK и платформы
- `practices` — workflows, recipes и how-to
- `cases` — практические примеры
- `comparisons` — сравнительные обзоры
- `articles` — переводы и конспекты статей
- `sources` — внешние источники и provenance

## Примеры использования

### Быстрое добавление статьи
```bash
/add-source url https://example.com/article.md
```

### Добавление с тегами
```bash
/add-source url https://llama.meta.com/ --tags llm,meta --category sources
```

### Добавление локального PDF
```bash
/add-source file ./papers/attention-is-all-you-need.pdf --category patterns
```

### Создание новой заметки
```bash
/add-source doc "Конспект: Transformer" --category patterns
```

## Валидация
- URL должен быть доступным (проверка HTTP 200)
- Файл должен существовать
- Название категории должно соответствовать существующей папке
- Теги должны быть валидными (без пробелов, спецсимволов)

## Интеграция с README.md и обзорными страницами
После добавления источника автоматически обновляется `README.md` или соответствующий `overview.md`:
- Добавляется запись в соответствующий раздел
- Сохраняется ссылка на файл
- Обновляется дата последнего изменения
