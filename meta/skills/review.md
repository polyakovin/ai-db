# Review — проверка изменений перед коммитом

## Когда использовать
После завершения изменений, перед коммитом.

## Алгоритм
1. Прочитай diff — каждая строка связана с задачей?
2. Проверь на минимальный diff: нет лишних форматирований, переименований, рефакторинга.
3. Проверь ссылки — нет ли битых Markdown-ссылок.
4. Запусти валидацию vault (см. [AGENTS.md](../../AGENTS.md#проверки)).
5. Если есть замечания — исправь или запиши в `meta/decisions.md`.

## Проверка
- diff связан с задачей.
- validate-vault.sh проходит (см. [AGENTS.md](../../AGENTS.md#проверки)).
- Нет битых ссылок.
- Нет лишних изменений.
